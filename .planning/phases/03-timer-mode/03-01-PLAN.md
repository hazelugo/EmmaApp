# Plan 3.1: Timer Math Data & useTimer Composable

## Objective
Extract the math problem generation logic so it can be shared between `useMathGame` and the new `useTimer`, then create the `useTimer` composable to manage the 60-second sprint challenge state.

## Context
- .planning/phases/03-timer-mode/03-CONTEXT.md
- src/composables/useMathGame.js
- src/composables/useTimer.js (NEW)
- src/utils/mathUtils.js (NEW)

## Tasks

<task type="auto">
  <name>Extract problem generator</name>
  <files>
    src/utils/mathUtils.js
    src/composables/useMathGame.js
  </files>
  <action>
    - Create `src/utils/mathUtils.js`.
    - Export a pure function `generateOperands(operator, maxOperand)` that returns `{ a, b }`.
    - Move the explicit generation logic (the `Math.random` math for '+', '-', '×', '÷') from `useMathGame.js` into this utility.
    - Refactor `generateProblem` in `useMathGame.js` to call `generateOperands` instead of doing the math inline.
  </action>
  <verify>npm run build # should verify the build after refactoring</verify>
  <done>Math logic is extracted to utility and useMathGame still functions identically.</done>
</task>

<task type="auto">
  <name>Create useTimer composable</name>
  <files>
    src/composables/useTimer.js
  </files>
  <action>
    - Create `src/composables/useTimer.js`.
    - Exports a reactive state: `isActive` (bool), `timeLeft` (number, starts at 60), `coins` (number), `correctCount` (number), `highScore` (read from localStorage key `emma-timer-best`).
    - Expose `startTimer(onComplete)` function: resets score/count/timeLeft, sets `isActive` to true, uses `setInterval` (1000ms), and calls `onComplete()` when it reaches 0.
    - Expose `stopTimer()` to clear the interval.
    - Expose `handleComplete()` to check if `correctCount > highScore`, save to `localStorage`, and mark `isActive` false.
    - Expose `incrementScore()` and `incrementCorrect()`.
  </action>
  <verify>npm run build</verify>
  <done>useTimer.js is created with all required state and functions, saving high score to 'emma-timer-best'.</done>
</task>

## Success Criteria
- [ ] `mathUtils.js` contains `generateOperands` for all 4 operators.
- [ ] `useMathGame` imports and uses `generateOperands`.
- [ ] `useTimer.js` controls the 60-second countdown lifecycle.
