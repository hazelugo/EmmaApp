## Typography

All text uses the `Gamtex` font family (`var(--font-display)` / `var(--font-body)`).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 | Item color name, price label, status badge text |
| Label | 16px (`text-base`) | 700 (extrabold) | 1.4 | Equipped/Owned/price badge, card metadata |
| Heading | 20px (`text-xl`) | 800 (extrabold) | 1.2 | Shop overlay title ("Star Shop") |
| Display | 28px (`text-3xl`) | 800 (extrabold) | 1.1 | Star balance shown in shop header area |

Weight palette: **400 (normal) + 700–800 (extrabold)**. Gamtex reads as bold by default; use `font-extrabold` for headings, `font-bold` for labels, no explicit weight class for body.

Source: existing components use `font-extrabold tracking-wide text-star-gold` for score display and `font-bold text-sm` for character name badges — carry forward exactly.

## Color

All colors reference existing `@theme` tokens from `src/style.css`. No new color values are introduced in this phase.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `mushroom-white` | `#FFF8F0` | Shop overlay background surface |
| Secondary (30%) | `sky / sky-top` | `#87CEEB / #4A90D9` | Shop overlay backdrop (same as `CharacterSelect` fixed overlay), item card background tint (`bg-sky/30`) |
| Accent (10%) | `star-gold` | `#FFD700` | Star price label, "Equipped" badge background, undo toast border |
| Destructive | `mario-red` | `#E52521` | Disabled/locked price indicator text only; no destructive actions exist in this phase |

Accent (`star-gold`) reserved for exactly:
1. Star cost price labels on item cards
2. "✓ Equipped" badge background on the equipped card
3. Undo toast border/highlight
4. Star balance display in the shop overlay header

Secondary semantic (`mario-red`) used for:
- Shop icon button in `ScoreHeader` (matches header's existing `bg-mario-red/95` surface — button sits within it, no new surface needed)
- Disabled/unaffordable price text dimmed to `text-mario-red/50`

Character-specific accent colors (peach, daisy, rosalina, mario-red) are used on item card borders and background tints, matching the pattern established in `CharacterSelect.vue`. Each variant card uses its character's `border-*` and `bg-*/60` tokens.

## Interaction Contract

### ShopOverlay

- **Open:** Tap shop icon in `ScoreHeader` → `App.vue` sets `showShop = true` → `<Transition name="fade">` reveals overlay
- **Close:** Tap ✕ button (top-right of overlay, `w-11 h-11` touch target) → `showShop = false`
- **Overlay stays open** after purchase and after undo window expires (D-08)
- Entry/exit animation: `fade` transition (same as `CharacterSelect.vue`)

### ShopItemCard interaction states

| State | Visual |
|-------|--------|
| Available (can afford) | Full opacity, price in `star-gold`, tap purchases immediately |
| Equipped | `star-gold` badge "✓ Equipped" overlaid bottom-left, ring highlight `border-star-gold` |
| Owned (not equipped) | "Owned" badge in `mushroom-white/90`, normal border |
| Unaffordable | Opacity 50% (`opacity-50`), price label dimmed `text-mario-red/50`, tap is no-op, no cursor feedback |

### Purchase flow

1. Tap affordable card → instant equip (optimistic, D-06): character variant updates in `MascotPanel` immediately
2. Stars deducted from `stars` ref in `useMathGame.js`
3. `UndoToast` appears at bottom of overlay with 10-second countdown
4. If undo tapped within 10s: stars restored, previous variant re-equipped, toast dismissed
5. If 10s expires: toast auto-dismisses, purchase finalized, localStorage written

### Equip animation

Brief CSS `pop-in` (0.4s, already defined in `style.css`) plays on the card when a variant is equipped. No additional animation library needed.

### UndoToast

- Position: fixed bottom of overlay, `mx-4 mb-4`, full-width minus padding
- Background: `bg-mushroom-white block-border` (matches card pattern)
- Countdown: live seconds display ("Undo (9s)", "Undo (8s)"…)
- Auto-dismiss at 0s with fade transition
- Only one toast exists at a time — subsequent purchases replace the previous toast (D-07)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (shop entry) | "Shop" (button label in ScoreHeader, accompanied by 🏪 emoji) |
| Shop overlay title | "Star Shop" |
| Empty state heading | "Earn more stars to unlock items!" |
| Empty state body | "Answer math problems to earn ⭐ stars, then come back to spend them." |
| Affordable item CTA | (no label — tap the card itself; state is self-evident from price display) |
| Equipped badge | "✓ Equipped" |
| Owned badge | "Owned" |
| Undo toast | "Undo (Ns)" where N counts down from 10 to 1 |
| Undo action label | "Undo" |
| Unaffordable tooltip | (none — dimmed state is self-evident for a 6-year-old audience; no tooltip added) |
| Error: purchase failed | "Oops! Not enough stars. Keep answering to earn more!" |
| Destructive actions | None in this phase (no delete, no reset). Undo reversal is not destructive — no confirmation required beyond the 10s window. |

Star price defaults (Claude's discretion per CONTEXT.md):

| Variant tier | Star price |
|--------------|-----------|
| Default (base) variant | 0 stars — free/already equipped at start |
| Tier-1 variant (any non-default) | 15 stars |

All 5 variants per character: 1 default (free) + 4 purchasable at 15 stars each = 15 stars max cost per variant, 300 stars total catalog value.

## Accessibility Contract

- All interactive cards: `role="button"` or native `<button>`, `aria-label="{Character} {ColorName} variant — {price} stars — {state}"`
- Shop open button: `aria-label="Open Star Shop"`
- Close button: `aria-label="Close shop"`
- Undo toast: `role="status"` (live region), `aria-live="polite"`
- Disabled cards: `aria-disabled="true"`, `tabindex="-1"`, `cursor-not-allowed`
- Reduced motion: `prefers-reduced-motion` already handled globally in `style.css` — `pop-in` animation will auto-disable
